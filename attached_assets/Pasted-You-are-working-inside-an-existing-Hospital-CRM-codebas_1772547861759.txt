- Cold
- Warm
- Warm+
- Warm++
- Hot
- Very Hot
- Dormant

Temperature Logic Rules:

Event → Impact
- Appointment Booked → Warm
- Rescheduled Once → No change
- Rescheduled 2+ times → Downgrade 1 level
- No Show → Downgrade 1 level
- Consultation Done → Warm+
- Estimate Shared → Warm++
- Insurance Approved → Hot
- Advance Received → Very Hot
- No activity > X days → Dormant

All temperature changes must:
- Be auto-computed
- Log previous → new value
- Log trigger event
- Store reference ID
- Write to timeline

SECTION 2 — AUTO HANDOVER ENGINE (CRITICAL)

Define stage-based ownership transfer.

Stage 0: Lead Created (Meta / Callyzer / Manual)
- owner_team = Telecalling
- primary_owner = auto assigned (round robin or load-based)
- SLA timer starts

Stage 1: Appointment Booked
- Handover to Front Office
- owner_team = Front Office
- primary_owner = Branch Front Office user
- Telecaller becomes read-only

Stage 2: Check-In (Front Office marks Checked In)
- Handover to Doctor
- owner_team = Doctor
- primary_owner = appointment.doctor_id
- Supporting owners = mapped Medical Assistant
- Front office view-only
- Telecaller removed from active visibility

Stage 3: Doctor Consultation Done
- If no Episode exists → Auto create Episode
- Link: patient_id + appointment_id + lead_id

Stage 4: Treatment Planning / Estimate Shared
- Handover to Financial Counsellor
- owner_team = Financial
- Doctor becomes supporting owner

Stage 5: Insurance Applicable
- Handover to Insurance Desk
- owner_team = Insurance

Stage 6: Surgery Scheduled
- Handover to OT / IP Desk

Stage 7: Discharge
- Handover to Post Care

Stage 8: Post Care Completed
- Activate Referral Engine

Create new table:
handover_log:
- entity_type
- entity_id
- from_user
- to_user
- from_team
- to_team
- trigger_event
- timestamp
