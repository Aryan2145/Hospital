We need to connect the CRM appointment booking event with WATI WhatsApp template API.

Objective:
Whenever a new appointment is successfully booked in the CRM, automatically send an approved WATI WhatsApp template message to the patient.

Use Case:
Appointment booking confirmation and appointment reminders through WhatsApp.

Important:
The appointment must be saved first. WhatsApp sending should happen only after the appointment is successfully created. If WATI message sending fails, the appointment should still remain booked. Only the WhatsApp message status should be marked as failed.

Approved WATI Templates:
1. viroc_appointment_confirmation
2. viroc_appointment_reminder

Template 1:
Template name: viroc_appointment_confirmation

Template variable mapping:
{{1}} = Patient name
{{2}} = Doctor name
{{3}} = Appointment date
{{4}} = Appointment time
{{5}} = Hospital name
{{6}} = Hospital contact number

Template 2:
Template name: viroc_appointment_reminder

Template variable mapping:
{{1}} = Patient name
{{2}} = Doctor name
{{3}} = Appointment date
{{4}} = Appointment time
{{5}} = Hospital name
{{6}} = Hospital contact number

WhatsApp receiver:
Patient mobile number with country code.

Example:
919876543210

Required CRM data fields:
- patient_name
- patient_mobile
- doctor_name
- appointment_date
- appointment_time
- hospital_name
- hospital_contact_number

Implementation Requirements:

1. Add WATI environment variables

Add these environment variables:

WATI_API_ENDPOINT=
WATI_API_TOKEN=

Example:
WATI_API_ENDPOINT=https://live-mt-server.wati.io/{tenant_id}
WATI_API_TOKEN=your_wati_bearer_token

Do not hardcode WATI token in the code.

2. Create WATI service

Create a reusable WATI service/function that sends template messages.

Function name suggestion:
sendWatiTemplateMessage()

It should accept:
- whatsappNumber
- templateName
- broadcastName
- parameters

It should call:

POST {WATI_API_ENDPOINT}/api/v2/sendTemplateMessage?whatsappNumber={whatsappNumber}

Headers:
Authorization: Bearer {WATI_API_TOKEN}
Content-Type: application/json

Body format for appointment confirmation:

{
  "template_name": "viroc_appointment_confirmation",
  "broadcast_name": "VIROC Appointment Confirmation",
  "parameters": [
    {
      "name": "patient_name",
      "value": "Patient Name"
    },
    {
      "name": "doctor_name",
      "value": "Doctor Name"
    },
    {
      "name": "appointment_date",
      "value": "Appointment Date"
    },
    {
      "name": "appointment_time",
      "value": "Appointment Time"
    },
    {
      "name": "hospital_name",
      "value": "Hospital Name"
    },
    {
      "name": "hospital_contact",
      "value": "Hospital Contact Number"
    }
  ]
}

Body format for appointment reminder:

{
  "template_name": "viroc_appointment_reminder",
  "broadcast_name": "VIROC Appointment Reminder",
  "parameters": [
    {
      "name": "patient_name",
      "value": "Patient Name"

3. Trigger confirmation after appointment creation

Find the backend function/API where a new appointment is created.

After the appointment is successfully saved in the database, call the WATI service using:

Template:
viroc_appointment_confirmation

Broadcast name:
VIROC Appointment Confirmation

Pseudo-flow:
- Create appointment in database
- Fetch/prepare patient, doctor, hospital, date, and time data
- Format patient mobile number with country code
- Send WATI template message using viroc_appointment_confirmation
- Save WhatsApp message log
- Return appointment booking success response

4. Add appointment reminder scheduler

Create a scheduled job that checks upcoming appointments and sends reminders through WATI.

Reminder template:
viroc_appointment_reminder

Broadcast name:
VIROC Appointment Reminder

Suggested scheduler:
Run every 15 minutes.

Reminder logic:
- Find appointments where appointment time is within the next 24 hours
- Appointment status is Booked / Confirmed
- reminder_24hr_sent_at is empty
- Send viroc_appointment_reminder template
- Save WhatsApp message log
- Update reminder_24hr_sent_at

Optional second reminder:
- Find appointments where appointment time is within the next 2 hours
- Appointment status is Booked / Confirmed
- reminder_2hr_sent_at is empty
- Send viroc_appointment_reminder template
- Save WhatsApp message log
- Update reminder_2hr_sent_at

5. Prevent duplicate WhatsApp messages

Add these fields to the appointment record if they do not exist:

confirmation_sent_at
reminder_24hr_sent_at
reminder_2hr_sent_at

Before sending confirmation:
If confirmation_sent_at is already filled, do not send again.

Before sending 24-hour reminder:
If reminder_24hr_sent_at is already filled, do not send again.

Before sending 2-hour reminder:
If reminder_2hr_sent_at is already filled, do not send again.

6. Create WhatsApp message log table/model

Create a table/model named:

whatsapp_message_logs

Minimum fields:
- id
- tenant_id
- appointment_id
- patient_id
- mobile_number
- template_name
- message_type
- status
- wati_response
- wati_local_message_id
- error_message
- sent_at
- created_at
- updated_at

Message types:
APPOINTMENT_CONFIRMATION
APPOINTMENT_REMINDER_24HR
APPOINTMENT_REMINDER_2HR

Possible status values:
PENDING
SENT
FAILED
DELIVERED
READ
REPLIED

7. Save WATI response

If WATI API succeeds:
- Create whatsapp_message_logs record with status SENT
- Save full WATI response as JSON
- Save localMessageId if available in the WATI response
- Update appointment.confirmation_sent_at or reminder field with current datetime based on message type

If WATI API fails:
- Create whatsapp_message_logs record with status FAILED
- Save error message
- Do not roll back the appointment booking

8. Mobile number formatting

Before calling WATI:
- Remove spaces, +, -, brackets
- If mobile number is 10 digits, prefix 91
- If mobile number already starts with 91, use as-is

Final format example:
919876543210

9. Date and time formatting

Send patient-friendly date and time format:

appointment_date example:
10 May 2026

appointment_time example:
11:30 AM

10. Add safe error handling

WATI failure should not break appointment booking.

Example:

try {
  send WATI message
} catch (error) {
  save failed WhatsApp log
}

11. Add basic test case

Test by booking one appointment and confirm:
- Appointment is created
- WATI API is called
- Patient receives WhatsApp confirmation
- WhatsApp log is created
- confirmation_sent_at is updated
- If WATI token is wrong, appointment still gets booked and WhatsApp log shows FAILED

Test reminder:
- Create an appointment due within 24 hours
- Confirm reminder scheduler sends viroc_appointment_reminder
- WhatsApp log is created
- reminder_24hr_sent_at is updated
- Duplicate reminder is not sent

Expected Final Result:
When an appointment is booked in CRM, the patient automatically receives the approved WATI template viroc_appointment_confirmation with correct patient name, doctor name, appointment date, appointment time, hospital name, and hospital contact number.

Before the appointment, the patient automatically receives the approved WATI template viroc_appointment_reminder based on scheduled reminder logic.

Do not change the existing appointment booking flow except adding this post-booking WATI trigger, reminder scheduler, duplicate prevention, and WhatsApp logging.
