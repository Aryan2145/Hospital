FEATURE (UI + Aggregation Only): Show Full Patient Journey on Lead Details Page
Context:
On Lead details page, current “Activity Timeline” shows lead-level events (appointments, WhatsApp, temperature, handover).
But the episode journey timeline (Consultation Done → Treatment Planning → … → Completed) is not visible here.
We need the Lead page to show the COMPLETE journey stitched together in a CRM-friendly way.

IMPORTANT:
- This is CRM, not HMS.
- Do NOT change workflow logic or transitions in this task.
- This task is ONLY to display full journey in Leads page using existing data.

ROLE LANGUAGE:
- Super Admin = Programmer team (system level)
- CRM Admin = Frontend hospital admin

========================================================
A) ADD 2 NEW UI SECTIONS ON LEAD DETAILS PAGE
========================================================

1) Section 1: “Journey Snapshot” (Top, above Activity Timeline)
Show:
- Lead Stage (existing)
- Latest Episode Stage (from most recent episode)
- Episode Count (already present)
- Revenue Probability (already present)
- Expected Revenue (from latest episode if available)
- Current Ownership/Team (existing)
- Quick status chip:
  - “Active” / “Completed” / “Discontinued” based on latest episode.status if episode exists else lead.status

2) Section 2: “Treatment Journey Timeline” (Embedded)
If lead has episodes:
- Display a compact card listing each episode (latest first):
  - Episode Name
  - Current Episode Stage (status)
  - Start date
  - Last updated
  - A “View Episode” button
- Under the latest episode card, show its Journey Timeline entries (episode_timeline / status-change logs)
  - Only key events (stage changes + discontinued + post care tasks + follow-up initiated)
  - Expand/collapse option: “Show full episode timeline”

========================================================
B) UNIFIED TIMELINE (MOST IMPORTANT)
========================================================
Create a “Unified Journey Timeline” that merges events from:
1) lead_activity_timeline (existing)
2) appointments linked to lead_id
3) episode timeline events for all episodes linked to this lead_id
4) post care tasks / follow-up tasks linked to episode_id (if available)

Rules:
- Merge and sort by timestamp descending.
- Each event shows a badge “Lead” / “Appointment” / “Episode” / “Post Care”
- For Episode events show:
  - Episode Name
  - From Stage → To Stage
  - Changed by
  - Timestamp
- For Post Care tasks show:
  - Task type (Physio / Dressing / Stitch removal etc.)
  - Home Visit / Hospital Visit
  - Assigned to
  - Due date + status

Keep UI simple: one combined list with filter chips:
[All] [Lead] [Appointments] [Episode] [Post Care]

========================================================
C) DATA / API REQUIREMENTS
========================================================
Add backend endpoint to fetch “Lead Journey Aggregation”:

GET /api/leads/:id/journey

Return:
{
  leadSummary: {...},
  episodes: [
    { id, name, status, startDate, updatedAt, expectedRevenue, probability }
  ],
  episodeTimelineEvents: [
    { episodeId, type:"STAGE_CHANGE", from, to, by, at, notes? }
  ],
  appointmentEvents: [...],
  leadEvents: [...],
  postCareTasks: [...]
}

If endpoints already exist separately, you can aggregate in frontend, but backend aggregation is preferred for performance.

========================================================
D) EDGE CASES
========================================================
1) Lead has NO episode:
- Show Journey Snapshot with “No episode started yet”
- Unified timeline shows only lead+appointment events

2) Lead has MULTIPLE episodes:
- Show episode list; default expands latest episode journey
- Unified timeline includes all episode events with episode name label

3) Performance:
- Limit unified timeline to latest 100 events with “Load more”
- Add indexes on episode_timeline(lead_id or episode_id) and tasks(episode_id)

========================================================
E) DELIVERABLE
========================================================
- Lead detail page displays full journey
- Episode journey is visible without navigating away
- Unified timeline shows complete story across lead → appointment → episode → post care
- No changes to workflow logic, only display/aggregation
