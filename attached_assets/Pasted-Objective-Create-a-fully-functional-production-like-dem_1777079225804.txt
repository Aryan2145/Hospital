Objective:
Create a fully functional, production-like demo tenant for a hospital CRM system to be used by the RGB team for live product demonstrations. The system must feel real, interconnected, and demo-ready without any additional setup.

---

STRICT DATA RULES

* No real doctors from Viroc
* No real patient data
* All data must be synthetic but realistic (Indian context)
* Avoid real-world identifiable overlap
* Patient mobile numbers must start with digit 5 (e.g., 5XXXXXXXXX)

---

Create a multi-branch structure:

* Main Hospital (Hub):
  RGB Demo Hospital – Modi Nagar

* Spoke Centers (Branches):

  * Kushal Nagar
  * Prashant Nagar

Requirements:

* Distribute doctors and staff across branches
* Ensure some doctors operate across multiple branches (for realism)
* Ensure patient flow across branches (e.g., consultation at spoke → surgery at hub)
